## Navigation

다음으로 Header 컴포넌트에서 navigation bar를 만들어 봅시다.

```javascript title="src/components/Header/index.js"
import React from "react";
import Logo from "./logo";
import Link from "next/link";

const Header = () => {
  return (
    <div className="w-full p-4 px-10 flex items-center justify-between">
      <Logo />
      <nav className="w-max py-3 px-8 border border-solid border-dark rounded-full font-medium capitalize flex items-center fixed top-6 right-1/2 translate-x-1/2 bg-light/80 backdrop-blur-sm z-50">
        <Link href="/" className="mr-2">
          Home
        </Link>
        <Link href="/about" className="mx-2">
          About
        </Link>
        <Link href="/contact" className="mx-2">
          Contact
        </Link>
        <button>dark</button>
      </nav>
      <div>social links</div>
    </div>
  );
};

export default Header;
```

dark 텍스트는 추후 다크모드 아이콘을 넣어 기능을 붙이고, social links 부분도 작성할 예정입니다.

nav 태그에 적용된 tailwindCSS 속성 중 몇 가지를 알아볼까요?

- border-dark: 테두리 색상을 dark로 설정합니다. 여기서 dark는 tailwindCSS에 내장된 색상이 아닌, 이전 포스팅에서 tailwind.config.js에 작성한 dark 색상입니다.
- capitalize: 요소 내의 텍스트를 자동으로 대문자화합니다. 각 단어의 첫 글자만 대문자로 변환됩니다.
- right-1/2: 오른쪽으로부터의 위치를 부모 요소의 50% 지점으로 설정합니다.
- translate-x-1/2: 요소를 수평 방향으로 자신의 너비의 50%만큼 이동시킵니다. 이는 요소를 중앙 정렬하는 데 자주 사용됩니다.
- bg-light/80: 요소의 배경 색상을 ligh로 설정하고, 투명도를 80%로 조정합니다. dark와 마찬가지로 tailwind.config.js에 작성한 light 색상입니다.
- backdrop-blur-sm: 요소 뒤의 배경에 약간의 블러 효과를 적용합니다.
- z-50 : 드래그 할 때, 다른 게시물들에 가려지지 않기 위해 넉넉히 z-50 적용한겁니다.

변경 된 부분을 알아봅시다.

- cx : 이전 포스팅에서 작성한 cx함수는 여러 className을 받고 그 중 참(유효한) 값만 필터링하여 하나의 문자열로 결합하는 유틸리티 함수입니다. 여기서는 cx함수를 사용해서 className과 "w-full h-auto" 클래스를 결합하는 역할입니다. 즉, 받아온 className에 추가적으로 "w-full h-auto"라는 tailwindCSS 클래스를 조합하여 사용합니다.
- `className={cx(className, "w-full h-auto")}`: 이는 cx 함수를 사용하여 외부에서 전달받은 className과 "w-full h-auto"라는 클래스를 결합합니다.
- `{...rest}`: 이 구문은 컴포넌트에 전달된 나머지 모든 props를 SVG 요소에 적용합니다.
- viewBox는 SVG 요소의 뷰포트 내에서 SVG 내용이 차지할 영역을 정의합니다. viewBox 속성은 "minX minY width height" 형식의 값으로, SVG 내부 좌표계에서 보여질 영역을 지정합니다. minX와 minY는 보여질 영역의 시작점을 나타냅니다.
  width와 height는 보여질 영역의 너비와 높이를 나타냅니다. viewBox를 사용하면, SVG가 다양한 크기로 확대 또는 축소되어도 그 내용이 적절히 조정되어 보여집니다.

- 예를 들어, viewBox="0 0 42 43"은 SVG 내용이 0,0에서 시작하여 가로 42, 세로 43의 크기로 표시됨을 의미합니다. 이렇게 설정함으로써, SVG 요소의 실제 크기가 변경되더라도 내부 그래픽은 viewBox에 정의된 비율로 유지됩니다.
