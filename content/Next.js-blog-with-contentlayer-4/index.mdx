### load latest posts

Home Cover Section에는 가장 최신의 글이 오도록 만들겁니다.

이전 포스팅에서, content/blog-one/index.mdx 파일을 보면 파일 상단에 아래처럼 작성했습니다.

```html
---
title: "Next.js blog with contentlayer: 4. Home cover section"
description: Let's create home cover section
image: "../../public/blogs/blog1.png"
publishedAt: "2024-03-26"
updatedAt: "2024-03-26"
author: "Baekki"
isPublished: true
tags:
  - web development
---
```

이는 해당 게시물의 메타데이터이며, 각 게시물들의 publishedAt 날짜 기준으로 내림차순 정률 후 최신 게시물을 Home Cover Section에 노출해봅시다.

먼저 날짜와 시간을 다루기 위한 `date-fns` 라이브러리를 설치해줍니다.

> npm i date-fns

이제 시간을 내림차순하여 비교하기 위해 src/utils/index.js 에 작성해봅시다.

```javascript title="src/utils/index.js"
import { compareDesc, parseISO } from "date-fns";

export const cx = (...className) => className.filter(Boolean).join(" ");

export const sortBlogs = (blogs) => {
  return blogs
    .slice()
    .sort((a, b) =>
      compareDesc(parseISO(a.publishedAt), parseISO(b.publishedAt))
    );
};
```

- sortBlogs 함수는 블로그 게시물 목록을 받아서 해당 포스트들을 날짜 순으로 내림차순하는 역할을 합니다. 각 게시물의 publishedAt을 기준으로 합니다.
- blogs 배열의 복사본을 만듭니다. .slice() 사용하여 원본 배열은 변경되지 않고, 새로운 배열이 생성됩니다.
- 생성된 배열을 .sort() 메서드로 정렬합니다. 두 개의 요소를 비교하는 콜백 함수를 인자로 받습니다.
- 콜백 함수에서는 a와 b 두 블로그 게시물의 publishedAt 속성값을 parseISO 함수를 통해 Date 객체로 변환합니다.
- compareDesc 함수를 사용하여 변환된 두 Date 객체를 비교합니다. 이 함수는 첫 번째 인자가 두 번째 인자보다 미래 날짜일 경우 -1, 과거 날짜일 경우 1, 동일한 날짜일 경우 0을 반환합니다. 따라서 이 함수의 결과를 기반으로 배열이 내림차순으로 정렬됩니다.

이제 sortBlogs 함수를 HomeCoverSection.js에 임포트하여 적용해봅시다.

```javascript title="src/components/Home/HomeCoverSection.js"
import { sortBlogs } from "@/src/utils";
import React from "react";

const HomeCoverSection = ({ blogs }) => {
  const sortedBlogs = sortBlogs(blogs);
  const blog = sortedBlogs[0];

  return <div>This is Home Cover Section</div>;
};

export default HomeCoverSection;
```

sortedBlogs는 blogs를 받아 publishedAt 기준으로 내림차순 정렬되어 있는 배열입니다. 그리고 blog는 정렬된 블로그 중 0번째, 즉 가장 최신의 글을 의미합니다.

### load image

blog-one 이미지를 불러와봅시다.

먼저 contentlayer.config.js에서 image 타입을 수정해주겠습니다.

```javascript title="contentlayer.config.js"
import { makeSource, defineDocumentType } from "contentlayer/source-files";

const Blog = defineDocumentType(() => ({
  name: "Blog",
  filePathPattern: "**/**/*.mdx",
  fields: {
    title: {
      type: "string",
      required: true,
    },
    publishedAt: {
      type: "date",
      required: true,
    },
    updatedAt: {
      type: "date",
      required: true,
    },
    description: {
      type: "string",
      required: true,
    },
    image: {
      type: "image",
    },
    isPublished: {
      type: "boolean",
      default: true,
    },
    author: {
      type: "string",
      required: true,
    },
    tags: {
      type: "list",
      of: { type: "string" },
    },
  },
  computedFields: {
    url: {
      type: "string",
      resolve: (doc) => `/blogs/${doc._raw.flattenedPath}`,
    },
  },
}));

export default makeSource({
  /* options */
  contentDirPath: "content",
  documentTypes: [Blog],
});
```

image type : string ➡️ image로 수정 후 다시 실행해보면 string 타입일 때 보다 이미지 넓이/높이, blurhashDataUrl 등 확인 할 수 있습니다.

위 데이터를 통해서 이미지를 표현해봅시다.

```javascript title="src/components/Home/HomeCoverSection.js"
import { sortBlogs } from "@/src/utils";
import Image from "next/image";
import React from "react";

const HomeCoverSection = ({ blogs }) => {
  const sortedBlogs = sortBlogs(blogs);
  const blog = sortedBlogs[0];

  return (
    <div className="w-full inline-block">
      <article className="flex flex-col justify-end mx-10 relative h-[85vh]">
        <Image
          src={blog.image.filePath.replace("../public", "")}
          placeholder="blur"
          blurDataURL={blog.image.blurhashDataUrl}
          alt={blog.title}
          fill
          className="w-full h-full object-center object-cover rounded-3xl"
        />
      </article>
    </div>
  );
};

export default HomeCoverSection;
```

- blog.image.filePath.replace("../public", "") : ../public 경로를 제거한 나머지 경로를 이미지의 소스로 사용합니다.
- placeholder : 이 속성은 이미지가 로드되기 전까지 표시할 임시 이미지의 형태를 정의합니다. "blur" 값을 가질 경우, blurDataURL에 지정된 데이터 URL을 사용하여 흐릿한 미리보기 이미지를 표시합니다.
- blurDataURL : 흐릿한 이미지의 데이터 URL입니다. blog.image.blurhashDataUrl을 사용하여, 이미지 로드 전에 표시될 흐릿한 이미지 데이터를 지정합니다.
- fill : 이미지가 부모 컨테이너를 완전히 채우도록 설정합니다.

const Tag = ({ link = "#", name, ...props }) => {
  return (

      <Link
        href={link}

- `const Tag = ({ link = "#", name, ...props })` : HomeCoverSection.js에서 prop을 전달 할때, 해당 태그에 관련된 카테고리 페이지로의 링크를 설정. 여기서 link = "#"은 기본 매개변수 값 설정입니다. HomeCoverSection.js에서 link의 prop이 전달되지 않았다면, 기본값인 "#"으로 인해 페이지 이동 없이 현재 페이지에 머무릅니다.
- ...props / cx() : Tag.js 컴포넌트의 코드에서 ...props는 컴포넌트에 전달된 모든 props 중에서 link와 name을 제외한 나머지 props를 나타냅니다. cx 함수는 "inline-block"과 props.className을 매개변수로 받아, 두 클래스를 조건부로 결합합니다.

### style

이제 title, description 스타일링과 함께 수정해보겠습니다.

```javascript title="src/components/Home/HomeCoverSection.js"
import { sortBlogs } from "@/src/utils";
import Image from "next/image";
import Link from "next/link";
import React from "react";
import Tag from "../Elements/Tag";

const HomeCoverSection = ({ blogs }) => {
  const sortedBlogs = sortBlogs(blogs);
  const blog = sortedBlogs[0];

  return (
    <div className="w-full inline-block">
      <article className="flex flex-col justify-end mx-10 relative h-[85vh]">
        <div className=" absolute top-0 left-0 bottom-0 right-0 h-full bg-gradient-to-b from-transparent from-0% to-dark/90 rounded-3xl z-0" />
        <Image
          src={blog.image.filePath.replace("../public", "")}
          placeholder="blur"
          blurDataURL={blog.image.blurhashDataUrl}
          alt={blog.title}
          fill
          className="w-full h-full object-center object-cover rounded-3xl -z-10"
        />
        <div className="w-3/4 p-16 flex flex-col justify-center text-light z-10">
          <Tag link={`/categories/${blog.tags[0]}`} name={blog.tags[0]} />
          <Link href={blog.url} className="mt-6">
            <h1 className="font-bold capitalize text-4xl">
              <span className="bg-gradient-to-r from-accent to-accent bg-[length:0px_6px] hover:bg-[length:100%_6px] bg-left-bottom bg-no-repeat transition-[background-size] duration-500">
                {blog.title}
              </span>
            </h1>
          </Link>
          <p className="inline-block mt-4 text-xl font-in">
            {blog.description}
          </p>
        </div>
      </article>
    </div>
  );
};

export default HomeCoverSection;
```

- `<Link href={blog.url} className="mt-6">` : blog.url은 contentlayer.config.js에서 computedFields 에서 정의한 url로 이동합니다. 즉, `http://localhost:3000/blogs/blog-one` 위 주소로 이동합니다.

위 h1 > span 태그에 적용된 속성 몇 가지를 살펴보면 아래와 같습니다.

- bg-gradient-to-r: 배경 그라데이션 방향을 왼쪽에서 오른쪽으로 설정합니다. 즉, 배경색이 왼쪽에서 오른쪽으로 변화합니다.
- from-accent to-accent: 그라데이션의 시작(from-accent)과 끝(to-accent) 색상을 설정합니다.
- bg-[length:0px_6px]: 배경 크기를 설정합니다. 여기서 0px는 너비, 6px는 높이를 의미하며, 초기 상태에서 배경이 표시되지 않음을 의미합니다.
- hover:bg-[length:100%_6px]: 호버 상태일 때의 배경 크기를 설정합니다. 100%는 너비를 의미하며, 6px는 높이를 유지합니다. 사용자가 요소 위에 마우스를 올렸을 때 배경이 요소의 전체 너비로 확장되도록 합니다.
- bg-left-bottom: 배경의 위치를 왼쪽 아래로 설정합니다.
- bg-no-repeat: 배경 이미지가 반복되지 않도록 합니다. 여기서는 그라데이션 효과에 적용되므로, 기본적으로 반복되지 않는 속성이지만, 명시적으로 반복을 방지하기 위해 사용됩니다.
- transition-[background-size]: 배경 크기 변경에 대한 전환 효과를 적용합니다. 이는 배경 크기가 변할 때(호버) 부드럽게 전환되도록 합니다.
- duration-500: 전환 효과의 지속 시간을 설정합니다.
