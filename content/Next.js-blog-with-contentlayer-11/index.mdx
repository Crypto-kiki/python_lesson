```sql title="SQL Editor"
-- grant create on schema public to anon
create
or replace function increment (slug_text text) returns void AS $$
declare view_id int;
begin

select id into view_id from views where slug = slug_text limit 1;

if view_id is not null then
update views set count = count + 1 where id = view_id;
else
insert into views (slug, count) values (slug_text, 1);
end if;
end;
$$ language plpgsql;
```

- `-- grant create on schema public to anon` : 이 줄은 public 스키마에 대한 생성 권한을 anon 사용자에게 부여합니다. 여기서 anon은 사용자 이름이며, 이 사용자는 보통 익명 사용자를 의미합니다. (함수와 직접적인 관련은 없음)
- `create or replace function increment (slug_text text) returns void AS $$` :
  - create or replace function : 새 함수를 생성하거나, 만약 같은 이름의 함수가 이미 존재한다면 그 함수를 대체합니다.
  - increment : 함수의 이름.
  - (slug_text text) : increment 함수가 받는 인자를 나타내며, slug_text라는 이름의 text 타입의 인자를 받습니다.
  - returns void : 이 함수가 반환값이 없음을 의미합니다.
  - AS $$ : 함수 본문의 시작을 나타냅니다.
- declare view_id int; : declare 구문은 함수 내에서 사용될 변수를 선언합니다. 여기서는 view_id라는 이름의 int 타입 변수를 선언합니다.
- begin : begin은 PL/pgSQL 함수에서 로직을 시작하는 키워드입니다.
- select id into view_id from views where slug = slug_text limit 1; : 이 구문은 views 테이블에서 slug 컬럼의 값이 인자로 받은 slug_text와 동일한 레코드의 id를 조회하여 view_id 변수에 저장합니다. limit 1은 결과가 여러 개일 경우에도 오직 하나의 결과만 가져오도록 제한합니다.
- if view_id is not null then : 이 if 조건문은 view_id 변수에 값이 할당되었는지 (즉, 조회된 레코드가 있는지) 확인합니다.
- update views set count = count + 1 where id = view_id; : 조건문이 참일 경우 (레코드가 있을 경우), views 테이블에서 해당 id를 가진 레코드의 count 값을 1 증가시킵니다.
- else <br/>
  insert into views (slug, count) values (slug_text, 1); : 조건문이 거짓일 경우 (레코드가 없을 경우), views 테이블에 새로운 레코드를 추가합니다. 이때 slug 컬럼에는 slug_text 값을, count 컬럼에는 1을 설정합니다.
- end if; : if 조건문의 끝을 나타냅니다.
- end; : 함수 본문의 끝을 나타내며, $$는 함수 정의의 끝을 표시합니다.
- $$ language plpgsql; : 이 함수가 PL/pgSQL 언어로 작성되었음을 나타냅니다.

> 즉, 위 함수는 특정 slug 값에 대한 조회수를 관리합니다. slug 값이 있는 경우 조회수를 증가시키고, 없는 경우 새로운 slug를 등록하면서 조회수를 1로 시작합니다.

>
> - .env는 프로젝트 전반에 걸쳐 기본적으로 사용될 환경 변수를 정의합니다.

> - 즉, 둘 다 중요한 설정 정보를 저장하지만, 범위와 적용 환경이 다름.

```javascript title="src/components/Blog/ViewCounter.js"
"use client";

import { createClientComponentClient } from "@supabase/auth-helpers-nextjs";
import { useEffect, useState } from "react";

const supabase = createClientComponentClient();

const ViewCounter = ({ slug, noCount = false, showCount = true }) => {
  const [views, setViews] = useState(0);

  useEffect(() => {
    const incrementView = async () => {
      try {
        let { error } = await supabase.rpc("increment", {
          slug_text: slug,
        });
        if (error) {
          console.log(
            "Error incrementing view count inside try block :",
            error
          );
        }
      } catch (error) {
        console.error(
          "This error occured while incrementing the view count:",
          error
        );
      }
    };

    if (!noCount) {
      incrementView();
    }
  }, [slug, noCount]);

  useEffect(() => {
    const getViews = async () => {
      try {
        let { data, error } = await supabase
          .from("views")
          .select("count")
          .match({ slug: slug })
          .single();

        if (error) {
          console.log(
            "Error incrementing view count inside try block :",
            error
          );
        }

        console.log(data);

        setViews(data ? data.count : 0);
      } catch (error) {
        console.error(
          "This error occured while incrementing the view count:",
          error
        );
      }
    };

    getViews();
  }, [slug]);

  if (showCount) {
    return <div>{views} views</div>;
  } else {
    return null;
  }
};

export default ViewCounter;
```
