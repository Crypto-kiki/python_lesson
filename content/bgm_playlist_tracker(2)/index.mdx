이번 교안의 핵심은 단순히 페이지를 몇 개 더 만드는 데 있지 않습니다.

| 함수형 CRUD 흐름 이해  | 생성(Create), 조회(Read), 수정(Update), 삭제(Delete) 전체 흐름을 직접 구현한다 |
| 상세페이지의 역할 이해    | 목록과 상세의 역할 차이를 분명히 이해한다                                     |
| 조건부 렌더링 경험      | 유튜브 링크가 있을 때만 iframe을 보여주는 조건부 렌더링을 구현한다                    |

# 4. forms.py 만들기

먼저 `Song` 모델 기반 입력 폼을 쉽게 만들기 위해 `ModelForm`을 사용하겠습니다.

```python title="music/forms.py"
from django import forms
from .models import Song


class SongForm(forms.ModelForm):
    class Meta:
        model = Song
        fields = ["artist", "title", "rating", "comment", "youtube_url"]
        widgets = {
            "artist": forms.TextInput(
                attrs={
                    "class": "form-input",
                    "placeholder": "가수명을 입력하세요",
                }
            ),
            "title": forms.TextInput(
                attrs={
                    "class": "form-input",
                    "placeholder": "곡 제목을 입력하세요",
                }
            ),
            "rating": forms.NumberInput(
                attrs={
                    "class": "form-input",
                    "min": 1,
                    "max": 5,
                    "placeholder": "1 ~ 5",
                }
            ),
            "comment": forms.Textarea(
                attrs={
                    "class": "form-textarea",
                    "rows": 5,
                    "placeholder": "이 곡을 좋아하는 이유를 적어보세요",
                }
            ),
            "youtube_url": forms.URLInput(
                attrs={
                    "class": "form-input",
                    "placeholder": "https://www.youtube.com/watch?v=...",
                }
            ),
        }
```

---

## 왜 ModelForm을 사용할까?

기존 모델이 이미 있으므로, 필드를 다시 일일이 폼으로 선언하는 대신
모델 기반으로 자동 폼을 생성하는 것이 훨씬 효율적입니다.

즉 아래처럼 생각하면 됩니다.

```
Song 모델이 이미 있음
→ SongForm이 그 구조를 활용함
→ HTML 입력 폼을 더 쉽게 만들 수 있음
```

---

## `fields`는 무엇을 의미할까?

```python title="music/forms.py"
fields = ["artist", "title", "rating", "comment", "youtube_url"]
```

이 코드는 어떤 필드를 사용자에게 입력받을지 정하는 부분입니다.

즉 사용자는 아래 항목을 브라우저에서 직접 입력하게 됩니다.

* 가수
* 곡 제목
* 별점
* 한줄 코멘트
* 유튜브 링크

`created_at`, `updated_at` 같은 필드는 자동 처리되므로 폼에 넣지 않습니다.

---

## `widgets`는 왜 사용할까?

`widgets`는 폼의 HTML 모양과 속성을 조금 더 보기 좋게 다듬기 위해 사용합니다.

예를 들면:

* input에 CSS 클래스 넣기
* placeholder 넣기
* textarea 행 수 지정하기
* rating 최소/최대 범위 힌트 주기

즉 이번 프로젝트에서는 단순 기능만이 아니라,
**폼 화면도 어느 정도 예쁘게 보여주기 위해 widgets를 같이 설정**합니다.

# 5. views.py에 CRUD View 작성하기

이제 `music/views.py`를 CRUD 흐름에 맞게 확장하겠습니다.

```python title="music/views.py"
from urllib.parse import parse_qs, urlparse

from django.shortcuts import get_object_or_404, redirect, render

from .forms import SongForm
from .models import Song



def extract_youtube_video_id(url):
    if not url:
        return None

    parsed_url = urlparse(url)

    if "youtu.be" in parsed_url.netloc:
        return parsed_url.path.lstrip("/")

    if "youtube.com" in parsed_url.netloc:
        query = parse_qs(parsed_url.query)
        if "v" in query:
            return query["v"][0]

        path_parts = parsed_url.path.split("/")
        if "embed" in path_parts:
            embed_index = path_parts.index("embed")
            if embed_index + 1 < len(path_parts):
                return path_parts[embed_index + 1]

    return None



def song_list(request):
    songs = Song.objects.all()
    return render(request, "music/song_list.html", {"songs": songs})



def song_create(request):
    if request.method == "POST":
        form = SongForm(request.POST)
        if form.is_valid():
            song = form.save()
            return redirect("music:song_detail", pk=song.pk)
    else:
        form = SongForm()

    return render(request, "music/song_form.html", {"form": form, "mode": "create"})



def song_detail(request, pk):
    song = get_object_or_404(Song, pk=pk)
    video_id = extract_youtube_video_id(song.youtube_url)
    embed_url = f"https://www.youtube.com/embed/{video_id}" if video_id else None

    context = {
        "song": song,
        "embed_url": embed_url,
    }
    return render(request, "music/song_detail.html", context)



def song_update(request, pk):
    song = get_object_or_404(Song, pk=pk)

    if request.method == "POST":
        form = SongForm(request.POST, instance=song)
        if form.is_valid():
            song = form.save()
            return redirect("music:song_detail", pk=song.pk)
    else:
        form = SongForm(instance=song)

    return render(request, "music/song_form.html", {"form": form, "mode": "update", "song": song})



def song_delete(request, pk):
    song = get_object_or_404(Song, pk=pk)

    if request.method == "POST":
        song.delete()
        return redirect("music:song_list")

    return render(request, "music/song_confirm_delete.html", {"song": song})
```

# 6. `extract_youtube_video_id()` 함수 이해하기

이번 2편에서 가장 새롭게 들어가는 부분은 바로 **유튜브 링크 처리**입니다.

우리는 상세페이지에서 영상을 재생하고 싶지만,
사용자가 입력하는 링크는 보통 아래처럼 다양합니다.

* `https://www.youtube.com/watch?v=VIDEO_ID`
* `https://youtu.be/VIDEO_ID`
* `https://www.youtube.com/embed/VIDEO_ID`

그런데 iframe에는 보통 이런 주소가 들어가야 합니다.

```text
https://www.youtube.com/embed/VIDEO_ID
```

즉, 원본 링크에서 `VIDEO_ID`만 추출해서 embed 주소로 바꿔줘야 합니다.

이번 함수는 바로 그 역할을 합니다.

```python title="music/views.py"
def extract_youtube_video_id(url):
```

이 함수는 링크를 받아서 `VIDEO_ID`만 꺼내고,
상세 View에서 그것을 다시 `embed_url`로 조합합니다.

---

## 왜 별도 함수로 분리할까?

그냥 `song_detail()` 안에 전부 써도 동작은 할 수 있습니다.
하지만 그렇게 하면 코드가 길어지고, 역할이 섞입니다.

그래서 아래처럼 분리하는 것이 더 좋습니다.

* `song_detail()` → 곡 상세페이지를 보여주는 역할
* `extract_youtube_video_id()` → 유튜브 링크에서 video id 추출 역할

이렇게 역할을 나누면 코드가 읽기 훨씬 쉬워집니다.

# 7. 등록 / 상세 / 수정 / 삭제 View 하나씩 이해하기

## 7-1. 곡 등록 View

```python title="music/views.py"
def song_create(request):
    if request.method == "POST":
        form = SongForm(request.POST)
        if form.is_valid():
            song = form.save()
            return redirect("music:song_detail", pk=song.pk)
    else:
        form = SongForm()

    return render(request, "music/song_form.html", {"form": form, "mode": "create"})
```

### 흐름 설명

```text
GET 요청
→ 빈 폼 보여주기

POST 요청
→ 사용자가 입력한 데이터 받기
→ 유효성 검사
→ 저장
→ 상세페이지로 이동
```

여기서 `form.save()`를 하면 `Song` 모델 객체가 실제 DB에 저장됩니다.

---

## 7-2. 곡 상세 View

```python title="music/views.py"
def song_detail(request, pk):
    song = get_object_or_404(Song, pk=pk)
```

상세페이지는 **특정 곡 한 개만 조회**해야 하므로 `pk`를 사용합니다.

즉 URL에서 받은 `pk`를 기준으로 해당 곡을 찾아오는 것입니다.

만약 존재하지 않는 `pk`면 자동으로 404 에러 페이지가 뜹니다.

---

## 7-3. 곡 수정 View

```python title="music/views.py"
def song_update(request, pk):
    song = get_object_or_404(Song, pk=pk)
```

수정도 먼저 기존 데이터를 가져와야 합니다.

그리고 중요한 부분은 아래입니다.

```python title="music/views.py"
form = SongForm(request.POST, instance=song)
```

이렇게 `instance=song`을 주면,
새 데이터를 만드는 것이 아니라 **기존 곡 정보를 수정하는 폼**이 됩니다.

---

## 7-4. 곡 삭제 View

```python title="music/views.py"
def song_delete(request, pk):
    song = get_object_or_404(Song, pk=pk)

    if request.method == "POST":
        song.delete()
        return redirect("music:song_list")
```

삭제는 GET 요청에서 바로 실행하지 않고,
보통 한 번 확인 페이지를 보여준 다음 POST로 삭제하는 것이 더 안전합니다.

즉 이번 교안에서는:

* GET → 삭제 확인 페이지 보여주기
* POST → 실제 삭제 실행

으로 처리합니다.

# 8. music 앱 URL 확장하기

이제 CRUD URL을 모두 연결하겠습니다.

```python title="music/urls.py"
from django.urls import path
from . import views

app_name = "music"

urlpatterns = [
    path("", views.song_list, name="song_list"),
    path("create/", views.song_create, name="song_create"),
    path("<int:pk>/", views.song_detail, name="song_detail"),
    path("<int:pk>/update/", views.song_update, name="song_update"),
    path("<int:pk>/delete/", views.song_delete, name="song_delete"),
]
```

---

## URL 흐름 해석

| URL             | 의미          |
| --------------- | ----------- |
| `/`             | 목록 페이지      |
| `/create/`      | 곡 등록 페이지    |
| `/<pk>/`        | 곡 상세 페이지    |
| `/<pk>/update/` | 곡 수정 페이지    |
| `/<pk>/delete/` | 곡 삭제 확인 페이지 |

이 구조는 Django CRUD에서 정말 자주 쓰이는 가장 기본적인 패턴입니다.

## 상세페이지가 왜 중요한가?

이 경우는 장고의 보안 설정 때문에 Referrer-Policy가 너무 강하게 잡혀 있습니다.

예를 들어 Django에서 SecurityMiddleware가 켜져 있으면,
기본적으로 referrer 정책이 보수적으로 동작해서
외부 iframe인 YouTube에 필요한 referrer가 안 넘어갈 수 있습니다.

settings.py 맨 아래에 설정값을 추가해주세요.

```python title="config/settings.py"
SECURE_REFERRER_POLICY = "strict-origin-when-cross-origin"
```

```html title="template/music/song_detail.html"
referrerpolicy="strict-origin-when-cross-origin"
```

# 17. 자주 만나는 오류 정리

## 1) `No module named 'music.forms'`

`music/forms.py` 파일을 아직 만들지 않았을 가능성이 큽니다.

반드시 아래 파일이 있어야 합니다.

```python title="music/forms.py"
from django import forms
from .models import Song
```

---

## 2) 수정 페이지에서 기존 값이 안 보임

대부분 아래 코드가 빠졌을 때 발생합니다.

```python title="music/views.py"
form = SongForm(instance=song)
```

또는 POST일 때도 아래처럼 `instance=song`이 있어야 합니다.

```python title="music/views.py"
form = SongForm(request.POST, instance=song)
```

---

## 3) 유튜브 영상이 안 나옴

가장 먼저 아래를 확인해보세요.

* 링크 형식이 올바른지
* `extract_youtube_video_id()` 함수가 제대로 작성되었는지
* `embed_url`을 context로 넘겼는지
* 템플릿에서 `{% if embed_url %}` 조건을 잘 썼는지

특히 `watch?v=` 형식과 `youtu.be/` 형식을 모두 처리하는지 확인해야 합니다.

---

## 4) `NoReverseMatch`

보통 URL name이 잘못됐을 때 발생합니다.

예를 들면 아래 name들을 정확히 맞춰야 합니다.

* `music:song_list`
* `music:song_create`
* `music:song_detail`
* `music:song_update`
* `music:song_delete`

---

## 5) 삭제 버튼 누르자마자 바로 삭제됨

삭제를 GET으로 처리했을 가능성이 있습니다.

이번 교안처럼 아래 구조로 가야 더 안전합니다.

* GET → 삭제 확인 화면
* POST → 실제 삭제

# 18. 이번 편 핵심 개념 요약

이번 2편에서 가장 중요한 개념은 아래입니다.

| 개념                  | 핵심                             |
| ------------------- | ------------------------------ |
| ModelForm           | 모델 기반으로 입력 폼을 효율적으로 만들 수 있다    |
| CRUD                | 생성, 조회, 수정, 삭제 전체 흐름을 직접 구현한다  |
| `get_object_or_404` | 특정 pk의 객체를 안전하게 조회할 수 있다       |
| `instance=`         | 수정 폼에서 기존 데이터를 불러올 때 사용한다      |
| 조건부 렌더링             | 유튜브 링크가 있을 때만 iframe을 보여줄 수 있다 |

즉 오늘은 아래 흐름을 직접 구현한 것입니다.

```
Form
→ Create
→ Detail
→ Update
→ Delete
→ Conditional Rendering
```
