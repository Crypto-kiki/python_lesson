## Jupyter Notebook이란?

Jupyter Notebook은 **코드를 셀(Cell) 단위로 실행하면서 결과를 바로 확인할 수 있는 도구**입니다.

공식 사이트는 아래에서 확인할 수 있습니다.

<a href="https://jupyter.org/" target="_blank">
  Jupyter 공식 홈페이지
</a>

일반적인 Python 파일 실행 방식은 보통 아래와 같습니다.

```text
코드 작성 → 파일 저장 → 전체 실행 → 결과 확인
```

반면 Jupyter Notebook은 아래처럼 사용할 수 있습니다.

```text
코드 작성 → 셀 하나만 실행 → 결과 확인 → 다음 셀 실행
```

즉, **한 줄씩 확인하면서 실험하기 쉬운 개발 환경**이라고 보면 됩니다.

특히 아래 같은 작업에서 매우 편합니다.

- 데이터 불러오기
- 데이터 확인하기
- 그래프 그리기
- 전처리 테스트
- 머신러닝 모델 실험
- 결과 비교 정리

## 왜 Jupyter Notebook을 사용할까?

처음 Python을 배울 때는 `.py` 파일만으로도 충분해 보일 수 있습니다.
하지만 AI, 데이터 분석, 머신러닝 쪽으로 가면 Jupyter Notebook의 장점이 매우 크게 느껴집니다.

### Jupyter Notebook 장점 요약

| 키워드 | 쉽게 설명하면 |
| --- | --- |
| 셀 단위 실행 | 코드 일부만 실행할 수 있어서 실험이 편합니다. |
| 결과 즉시 확인 | `print()`를 많이 쓰지 않아도 결과를 바로 볼 수 있습니다. |
| 시각화에 강함 | 그래프를 그리면 바로 아래에 출력됩니다. |
| 문서화 가능 | Markdown 셀로 설명을 함께 적을 수 있습니다. |
| 데이터 분석에 특화 | `pandas`, `scikit-learn`, `matplotlib`와 잘 어울립니다. |
| 실험 기록에 유리 | 어떤 순서로 실험했는지 파일에 남기기 좋습니다. |

예를 들어 아래처럼 **설명 + 코드 + 결과**를 한 파일 안에서 같이 관리할 수 있습니다.

```text
[설명]
이 셀에서는 데이터를 불러옵니다.

[코드]
import pandas as pd

[결과]
표 형태 데이터 출력
```

이 구조 때문에 Jupyter Notebook은 단순 코드 실행 도구가 아니라,
**실습 노트 + 실험 기록장 + 분석 리포트** 역할까지 함께 해줍니다.

## 프로젝트 준비

이번 교안에서는 `uv`를 사용하여 프로젝트를 세팅합니다.

먼저 프로젝트 폴더를 생성합니다.

```bash title="terminal"
mkdir scikit_learn
cd scikit_learn
code -r .
```

위 명령어의 의미는 다음과 같습니다.

- `mkdir scikit_learn` : 실습용 폴더 생성
- `cd scikit_learn` : 생성한 폴더로 이동
- `code -r .` : 현재 폴더를 VSCode로 열기

## 가상환경 생성

Python 실습에서는 **가상환경을 사용하는 습관**이 매우 중요합니다.
프로젝트마다 설치 패키지가 다를 수 있기 때문입니다.

```bash title="terminal"
uv venv
source .venv/bin/activate
```

가상환경이 잘 활성화되었는지 확인하려면 아래 명령어를 실행합니다.

```bash title="terminal"
which python
```

예를 들어 아래와 비슷한 경로가 보이면 정상입니다.

```bash title="result"
/Users/your-name/scikit_learn/.venv/bin/python
```

즉, 지금 사용하는 Python이 **전역 Python이 아니라 현재 프로젝트의 가상환경 Python**이라는 뜻입니다.

## Jupyter Notebook 설치

이제 Jupyter Notebook을 설치합니다.

```bash title="terminal"
uv pip install notebook
```

이 명령으로 Jupyter Notebook 실행에 필요한 패키지들이 함께 설치됩니다.

설치가 끝났다면 이제 Notebook을 실행할 준비가 완료된 것입니다.

## Jupyter Notebook 실행

운영체제나 환경에 따라 실행 방식이 약간 다릅니다.

### macOS

```bash title="terminal"
jupyter notebook
```

보통 브라우저가 자동으로 열립니다.

### WSL / Linux

```bash title="terminal"
jupyter notebook --no-browser --port=8888
```

이 명령은 브라우저를 자동으로 열지 않고,
Jupyter 서버만 실행합니다.

터미널에 아래와 비슷한 주소가 출력됩니다.

```text
http://localhost:8888/tree?token=65342e7337763d3b6ebbfe5b2351a513560534e33725bec6
http://127.0.0.1:8888/tree?token=65342e7337763d3b6ebbfe5b2351a513560534e33725bec6
```

이 주소를 브라우저에 붙여넣으면 됩니다.

## WSL에서 접속하는 방법

WSL에서는 브라우저가 자동으로 안 열릴 수 있습니다.
이때는 아래 두 가지 방식 중 하나로 접속하면 됩니다.

### 방법 1. URL 전체를 그대로 복사해서 접속

터미널에 출력된 주소 전체를 복사해서 브라우저에 붙여넣습니다.

```text
http://localhost:8888/tree?token=xxxxxxxxxxxxxxxx
```

### 방법 2. 토큰만 복사해서 로그인 화면에 입력

로그인 화면이 뜨면,
터미널 출력에서 `token=` 뒤의 긴 문자열만 복사합니다.

예시:

```text
b1944f390af040fce95ddd948b3f0b199528750e2cc37960
```

그리고 로그인 화면의 **Password or token** 입력칸에 붙여넣고 로그인합니다.

## 현재 실행 중인 Jupyter 서버 확인

Jupyter Notebook이 어디에서 실행 중인지 확인하고 싶다면 아래 명령어를 사용할 수 있습니다.

```bash title="terminal"
jupyter server list
```

예를 들어 아래처럼 출력될 수 있습니다.

```text
Currently running servers:
http://localhost:8888/?token=xxxxxxxxxxxx :: /Users/your-name/scikit_learn
```

이 정보로

- 어떤 포트에서 실행 중인지
- 어떤 프로젝트 폴더에서 실행 중인지
- 접속 주소가 무엇인지

확인할 수 있습니다.

## VSCode 확장 설치

VSCode에서 `.ipynb` 파일을 더 편하게 다루려면 Jupyter 관련 확장을 설치하는 것이 좋습니다.

추천 확장은 다음과 같습니다.

| 확장명 | 필요 여부 | 설명 |
| --- | --- | --- |
| Jupyter | 필수 | Jupyter Notebook 파일 실행 및 편집 핵심 기능 |
| Jupyter Keymap | 추천 | Jupyter 스타일 단축키 사용 가능 |
| Jupyter Notebook Renderers | 추천 | 시각화 결과를 더 잘 렌더링 |
| Jupyter Cell Tags | 선택 | 셀 태그가 필요할 때 사용 |
| Jupyter Slide Show | 선택 | 발표용 슬라이드 기능 |

가장 중요한 것은 **Jupyter 확장 하나**입니다.
처음에는 이것만 설치해도 충분합니다.

## 새 Notebook 파일 만들기

Jupyter 화면에서 새 파일을 만들 때는 보통 아래 순서로 진행합니다.

```text
New → Python 3
```

그러면 새로운 Notebook 파일이 생성됩니다.

파일명 예시:

```text
test.ipynb
```

`.ipynb`는 Jupyter Notebook 파일 확장자입니다.

## 첫 번째 테스트 셀 실행

이제 가장 간단한 테스트를 해보겠습니다.

아래 코드를 입력합니다.

```python title="test.ipynb"
print("Hello Jupyter")
```

실행 후 아래처럼 출력되면 정상입니다.

```text
Hello Jupyter
```

이 단계가 중요한 이유는,
지금 내 환경에서 **Jupyter가 실제로 정상 실행되는지 확인하는 첫 체크포인트**이기 때문입니다.

## 숫자 계산 테스트

이번에는 간단한 계산도 해보겠습니다.

```python title="test.ipynb"
10 + 20
```

결과:

```text
30
```

Jupyter Notebook에서는 마지막 표현식의 결과를 자동으로 보여주는 경우가 많습니다.
그래서 `print()` 없이도 값이 출력될 수 있습니다.

## 변수 테스트

```python title="test.ipynb"
a = 10
b = 20

print(a + b)
```

결과:

```text
30
```

이 실습은 아주 간단하지만,
앞으로 데이터 분석이나 머신러닝을 할 때도 결국은 이런 식으로

- 값 저장
- 계산
- 결과 확인

흐름이 반복됩니다.

## 문자열 테스트

```python title="test.ipynb"
name = "Baekki"
print("Hello,", name)
```

결과:

```text
Hello, Baekki
```

숫자뿐 아니라 문자열도 잘 처리되는지 확인해보는 테스트입니다.

## 리스트 테스트

```python title="test.ipynb"
numbers = [1, 2, 3, 4, 5]
print(numbers)
print(numbers[0])
```

결과:

```text
[1, 2, 3, 4, 5]
1
```

이 실습은 이후 `numpy`, `pandas`로 넘어가기 전에,
Python 기본 자료형을 Notebook에서 어떻게 다루는지 익히는 용도입니다.

## 간단한 데이터 분석 맛보기

이제 Jupyter Notebook이 왜 데이터 분석에 좋은지 아주 간단히 확인해보겠습니다.

먼저 `pandas`를 설치합니다.

```bash title="terminal"
uv pip install pandas
```

그 다음 Notebook 셀에 아래 코드를 입력합니다.

```python title="test.ipynb"
import pandas as pd

data = {
    "name": ["Alice", "Bob", "Chris"],
    "score": [95, 88, 92]
}

df = pd.DataFrame(data)
df
```

결과는 표처럼 출력됩니다.

이것이 Jupyter Notebook의 큰 장점 중 하나입니다.
단순 텍스트가 아니라 **보기 좋은 표 형태**로 결과를 바로 확인할 수 있습니다.

## 그래프 맛보기

그래프도 한번 그려보겠습니다.

먼저 설치:

```bash title="terminal"
uv pip install matplotlib
```

실습 코드:

```python title="test.ipynb"
import matplotlib.pyplot as plt

scores = [95, 88, 92]

plt.plot(scores)
plt.title("Score Graph")
plt.show()
```

이 코드를 실행하면 셀 아래에 그래프가 바로 나타납니다.

즉, Jupyter Notebook은

- 코드 작성
- 데이터 확인
- 표 보기
- 그래프 확인

을 한 화면에서 모두 할 수 있는 환경입니다.

## Jupyter Notebook 단축키

Notebook을 자주 사용할수록 단축키를 알아두는 것이 매우 중요합니다.

### 셀 추가 / 삭제

```text
A        : 위에 셀 추가
B        : 아래에 셀 추가
D, D     : 셀 삭제
Z        : 방금 삭제한 셀 되살리기
```

### 셀 타입 변경

```text
Y        : 코드 셀로 전환
M        : 마크다운 셀로 전환
```

### 복사 / 붙여넣기 / 잘라내기

```text
C        : 셀 복사
V        : 복사한 셀 아래에 붙이기
Shift+V  : 복사한 셀 위에 붙이기
X        : 셀 잘라내기
```

### 셀 실행

```text
Shift + Enter : 셀 실행 + 다음 셀 이동
Ctrl  + Enter : 셀 실행만
Alt   + Enter : 셀 실행 + 아래 새 셀 생성
```

처음에는 다 외우려고 하지 말고,
최소한 아래 세 개만 먼저 익혀도 충분합니다.

```text
A
B
Shift + Enter
```

이 세 개만 익혀도 실습 속도가 많이 빨라집니다.

## Jupyter 종료

작업을 마쳤다면 Jupyter 서버를 종료해야 합니다.

정상 종료:

```bash title="terminal"
jupyter notebook stop
```

강제 종료가 필요한 경우:

```bash title="terminal"
pkill -f ipykernel_launcher
```

강제 종료는 보통

- 창은 닫았는데 서버가 남아 있을 때
- 포트 충돌이 날 때
- 이전 Jupyter 프로세스가 꼬였을 때

사용합니다.

## 초보자가 자주 겪는 실수

### 1. 가상환경을 활성화하지 않고 설치함

이 경우 Jupyter가 전역 환경에 설치되어,
프로젝트별 패키지 관리가 꼬일 수 있습니다.

항상 먼저 아래를 확인하세요.

```bash title="terminal"
source .venv/bin/activate
which python
```

### 2. WSL에서 브라우저가 자동으로 안 열림

정상입니다.
WSL에서는 보통 URL을 직접 복사해서 Windows 브라우저에 붙여넣으면 됩니다.

### 3. `zsh: no matches found` 같은 에러

이건 명령어에 대괄호가 포함될 때 자주 발생합니다.
예를 들어 패키지 extras 설치 시,

```bash title="wrong"
uv pip install u8darts[torch]
```

처럼 입력하면 zsh가 먼저 해석해서 문제가 날 수 있습니다.
이럴 때는 따옴표를 사용합니다.

```bash title="correct"
uv pip install "u8darts[torch]"
```

### 4. 포트 충돌

8888 포트가 이미 사용 중이면 다른 포트를 사용하면 됩니다.

```bash title="terminal"
jupyter notebook --no-browser --port=9999
```

## 실습 체크리스트

아래 항목을 하나씩 확인해보세요.

- `uv venv`로 가상환경을 만들었다.
- `source .venv/bin/activate`로 활성화했다.
- `uv pip install notebook`으로 Jupyter를 설치했다.
- `jupyter notebook` 또는 `jupyter notebook --no-browser --port=8888`로 실행했다.
- 브라우저에서 정상 접속했다.
- `print("Hello Jupyter")` 테스트를 성공했다.
- 간단한 계산 테스트를 했다.
- `pandas` 표 출력까지 확인했다.
- 그래프 출력까지 확인했다.
