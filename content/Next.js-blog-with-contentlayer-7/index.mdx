## Blog page

### dynamic routes

먼저 각 게시물마다 url을 아래와 생성되도록 하려고합니다.

> http://localhost:3000/blogs/Next.js-blog-with-contentlayer-1

즉, `blog주소/blogs/게시물타이틀` 과 같이 생성되도록 하려면, 다이나믹 라우팅 사용해야 합니다.

참고링크 : <a href="https://nextjs.org/docs/app/building-your-application/routing/dynamic-routes" target="_blank">Next.js Dynamic Routes</a>

Next.js의 다이나믹 라우팅 기능은 사이트 내에서 다양한 경로를 동적으로 생성할 수 있게 해줍니다.

먼저, src/app/blogs/[slug]/page.js 를 생성합니다.

```javascript title="src/app/blogs/[slug]/page.js"
export default function BlogPage({ params }) {
  return <div>My Post: {params.slug}</div>;
}
```


여기서 slug는 URL에서 특정 페이지를 식별하는 데 사용되는 문자열이며, app/blogs/[slug]/page.js 파일 경로에 정의된 동적 경로의 일부입니다.

위 파일을 저장 후, 게시글을 클릭해보면 아래와 같이 url이 생성됩니다.

```javascript title="src/app/blogs/[slug]/page.js"
import { allBlogs } from ".contentlayer/generated";
import Tag from "@/src/components/Elements/Tag";
import Image from "next/image";

export default function BlogPage({ params }) {
  const blog = allBlogs.find((blog) => blog._raw.flattenedPath === params.slug);

  return (
    <article>
      <div className="mb-8 text-center relative w-full h-[70vh] bg-dark">
        <div className="w-full z-10 flex flex-col items-center justify-center absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2">
          <Tag
            name={blog.tags[0]}
            link={`/categories/${blog.tags[0]}`}
            className="px-6 text-sm py-2"
          />
          <h1 className="inline-block mt-6 font-semibold capitalize text-light text-5xl leading-normal relative w-5/6">
            {blog.title}
          </h1>
        </div>
        <div className="absolute top-0 left-0 right-0 bottom-0 h-full bg-dark/60" />
        <Image
          src={blog.image.filePath.replace("../public", "")}
          placeholder="blur"
          blurDataURL={blog.image.blurhashDataUrl}
          alt={blog.title}
          width={blog.image.width}
          height={blog.image.height}
          className=" aspect-square w-full h-full object-cover object-center"
        />
      </div>
    </article>
  );
}
```


- `const blog = allBlogs.find((blog) => blog._raw.flattenedPath === params.slug);` : .find 메소드를 사용하여 allBlogs 배열 내의 각 블로그 객체(blog)를 순회합니다. 순회하는 동안 각 블로그 객체의 \_raw.flattenedPath 프로퍼티가 현재 페이지의 URL에서 추출된 slug 값과 일치하는지 확인합니다. 이는 다이나믹 라우팅을 통해 접근한 현재 페이지의 경로를 기반으로 해당 페이지에 맞는 블로그 포스트 데이터를 찾기 위함입니다.

좌측 하단에 시간(publishedAt), views를 확인 할 수 있습니다. (views는 supabase를 적용할 예정입니다.)

reading time을 적용하기 위해서는 해당 라이브러리를 설치해야 합니다.

> npm i reading-time

그리고 contentlayer.config.js에 readingTime 필드를 추가합니다.

```javascript title="contentlayer.config.js"
import { makeSource, defineDocumentType } from "contentlayer/source-files";
import readingTime from "reading-time";

const Blog = defineDocumentType(() => ({
  name: "Blog",
  filePathPattern: "**/**/*.mdx",
  fields: {
    title: {
      type: "string",
      required: true,
    },
    publishedAt: {
      type: "date",
      required: true,
    },
    updatedAt: {
      type: "date",
      required: true,
    },
    description: {
      type: "string",
      required: true,
    },
    image: {
      type: "image",
    },
    isPublished: {
      type: "boolean",
      default: true,
    },
    author: {
      type: "string",
      required: true,
    },
    tags: {
      type: "list",
      of: { type: "string" },
    },
  },
  computedFields: {
    url: {
      type: "string",
      resolve: (doc) => `/blogs/${doc._raw.flattenedPath}`,
    },
    readingTime: {
      type: "json",
      resolve: (doc) => readingTime(doc.body.raw),
    },
  },
}));

export default makeSource({
  /* options */
  contentDirPath: "content",
  documentTypes: [Blog],
});
```


- readingTime : 주어진 텍스트의 길이를 기반으로 해당 텍스트를 읽는 데 걸리는 예상 시간을 계산하는 라이브러리입니다.
- type: "json" : JSON 타입이란, 예상 읽는 시간을 포함한 정보가 객체 형태로 반환될 것임을 의미합니다.
- `resolve: (doc) => readingTime(doc.body.raw)` : 현재 문서(doc)를 인자로 받으며, readingTime 라이브러리를 사용하여 doc.body.raw (즉, 블로그 포스트의 원시 본문 텍스트)를 분석합니다. readingTime 함수는 텍스트를 분석하여 읽는 데 걸리는 시간을 예측하고, 이 정보를 객체 형태로 반환합니다.

```javascript title="src/components/Blog/RenderMdx.js"
import React from "react";
import { useMDXComponent } from "next-contentlayer/hooks";

const RenderMdx = ({ blog }) => {
  const MDXContent = useMDXComponent(blog.body.code);

  return (
    <div>
      <MDXContent />
    </div>
  );
};

export default RenderMdx;
```


- useMDXComponent : next-contentlayer/hooks에서 가져온 훅으로, MDX 내용을 React 컴포넌트로 변환하는 데 사용됩니다.

하지만 위 코드를 실행하면 아래와 같은 오류가 발생합니다.

> Error: Expected component `Image` to be defined: you likely forgot to import, pass, or provide it.

이 오류는 MDX 파일 내에서 Image 컴포넌트를 사용하려고 했지만, 해당 컴포넌트를 정의하거나 제공하지 않아 발생하는 문제입니다.

RenderMdx 컴포넌트에서 MDXContent 컴포넌트를 사용할 때, MDX 파일 내에서 사용된 컴포넌트들에 대한 참조를 제공해야 합니다. 따라서 Image 컴포넌트를 RenderMdx 컴포넌트에 포함시키고, MDXContent에 이를 전달해야 합니다.

먼저, Next.js의 Image 컴포넌트를 import 합니다. 그런 다음, MDX 파일 내에서 사용할 수 있도록 MDXContent 컴포넌트에 해당 Image 컴포넌트를 전달합니다.

```javascript title="src/components/Blog/RenderMdx.js"
import React from "react";
import { useMDXComponent } from "next-contentlayer/hooks";
import Image from "next/image";

// MDX 파일에서 사용할 컴포넌트들을 정의합니다.
const mdxComponents = {
  Image, // Image 컴포넌트를 추가합니다.
};

const RenderMdx = ({ blog }) => {
  const MDXContent = useMDXComponent(blog.body.code);

  return (
    <div>
      <MDXContent components={mdxComponents} />
    </div>
  );
};

export default RenderMdx;
```


위 처럼 적용하면 아래와 같은 오류가 발생합니다.

> Error: Cannot access Image.propTypes on the server. You cannot dot into a client module from a server component. You can only pass the imported name through.

위 오류의 원인은 서버 사이드에서 실행될 때 Next.js의 Image 컴포넌트의 속성에 접근하려고 시도했기 때문에 발생합니다.

따라서 "use client" 지시어를 사용해줍니다.

```javascript title="src/components/Blog/RenderMdx.js"
"use client";

import React from "react";
import { useMDXComponent } from "next-contentlayer/hooks";
import Image from "next/image";

// MDX 파일에서 사용할 컴포넌트들을 정의합니다.
const mdxComponents = {
  Image, // Image 컴포넌트를 추가합니다.
};

const RenderMdx = ({ blog }) => {
  const MDXContent = useMDXComponent(blog.body.code);

  return (
    <div>
      <MDXContent components={mdxComponents} />
    </div>
  );
};

export default RenderMdx;
```

마크다운(mdx)에서 가져온 코드를 typography 요소의 스타일을 쉽게 관리하기 위해 tailwindCSS의 typography플러그인을 알아봅시다.

참고링크 : <a href="https://tailwindcss.com/docs/plugins#typography" target="_blank">TailwindCSS typography</a>

먼저 플러그인을 설치합니다.

> npm install -D @tailwindcss/typography

설치 후 tailwindcss.config.js에 플러그인 추가를 합니다.

```javascript title="tailwindcss.config.js"
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/components/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/app/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {
      colors: {
        dark: "#1b1b1b",
        light: "#fff",
        accent: "#7b00d3",
        accentDark: "#ffdb4d",
        gray: "#747474",
      },
      fontFamily: {
        mr: ["var(--font-mr)"],
        in: ["var(--font-in)"],
      },
    },
  },
  plugins: [require("@tailwindcss/typography")],
};
```

```javascript title="src/components/Blog/RenderMdx.js"
"use client";
```

```javascript title="contentlayer.config.js"
import { makeSource, defineDocumentType } from "contentlayer/source-files";
import readingTime from "reading-time";
import remarkGfm from "remark-gfm";

const Blog = defineDocumentType(() => ({
  name: "Blog",
  filePathPattern: "**/**/*.mdx",
  contentType: "mdx",
  fields: {
    title: {
      type: "string",
      required: true,
    },
    publishedAt: {
      type: "date",
      required: true,
    },
    updatedAt: {
      type: "date",
      required: true,
    },
    description: {
      type: "string",
      required: true,
    },
    image: {
      type: "image",

    isPublished: {
      type: "boolean",
      default: true,
    },
    author: {
      type: "string",
      required: true,
    },

      type: "list",
      of: { type: "string" },
```

다음으로 rehype 관련 라이브러리를 설치합니다.

> npm i rehype-slug

- rehype-slug : HTML 문서의 제목 요소에 고유한 ID를 자동으로 추가합니다. rehype-autolink-headings와 함께 사용될 때 매우 유용합니다. 제목에 ID를 부여함으로써, 개별 제목에 직접 링크를 걸거나 페이지 내에서 쉽게 참조할 수 있게 됩니다.

> npm i rehype-autolink-headings

- rehype-autolink-headings: 이 플러그인은 HTML 문서 내의 제목(heading) 요소에 자동으로 링크를 추가합니다. 사용자가 제목을 클릭하면 해당 제목의 위치로 쉽게 이동할 수 있게 되며, 주로 문서의 내비게이션을 향상시키기 위해 사용됩니다. Markdown 파일을 HTML로 변환한 뒤, 각 제목에 고유한 ID를 부여하고 이 ID를 이용해 링크를 생성합니다.

```javascript title="contentlayer.config.js"
import { makeSource, defineDocumentType } from "contentlayer/source-files";
```

```javascript title="contentlayer.config.js"
import rehypeAutolinkHeadings from "rehype-autolink-headings";
import rehypePrettyCode from "rehype-pretty-code";

import remarkGfm from "remark-gfm";

const Blog = defineDocumentType(() => ({
  name: "Blog",
  filePathPattern: "**/**/*.mdx",
  contentType: "mdx",
  fields: {
    title: {
      type: "string",
      required: true,
    },
    publishedAt: {
      type: "date",
      required: true,
    },
    updatedAt: {
      type: "date",
      required: true,
    },
    description: {
      type: "string",
      required: true,
    },
    image: {
      type: "image",
    },
    isPublished: {
      type: "boolean",
      default: true,
    },
    author: {
      type: "string",
      required: true,
    },
    tags: {
      type: "list",
      of: { type: "string" },
    },
  },
  computedFields: {
    url: {
      type: "string",
      resolve: (doc) => `/blogs/${doc._raw.flattenedPath}`,
    },
    readingTime: {
      type: "json",
      resolve: (doc) => readingTime(doc.body.raw),
    },
  },
}));

const codeOptions = {
  theme: "github-dark",
};

export default makeSource({
  /* options */
```
