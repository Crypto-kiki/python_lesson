## RPC failed?

찾아보니 다양한 원인이 있는 것 같습니다.

1. 가장 먼저 깃 레포의 주소가 맞는지 확인해봅시다.

> git remote -v

2. 깃 버전 업데이트 확인

3. 캐시 삭제 및 푸시 재시도

.git/ 디렉토리 내의 index.lock 파일이 남아있는지 확인하고, 있으면 삭제한 후 다시 시도해보세요.

> rm -f .git/index.lock

4. 네트워크 문제 확인. VPN 사용시, VPN을 비활성화하고 시도해보세요.

위의 방법들을 시도해봤지만, 위 문제들이 아니였습니다.

## http.postBuffer

Git에서 원격 저장소로 데이터를 푸시할 때, 데이터를 작은 청크(chunk)로 나누어 전송합니다.

http.postBuffer는 이러한 청크의 최대 크기를 정의하는 설정입니다. 기본적으로 이 크기는 1MB(1,048,576 바이트)로 설정되어 있습니다. 그러나 전송하는 데이터가 크거나 네트워크 상태가 불안정할 경우, Git이 데이터를 올바르게 전송하지 못하고 RPC failed 오류가 발생할 수 있습니다.

이 경우, http.postBuffer의 크기를 늘려줌으로써 한 번에 더 큰 청크를 전송할 수 있게 되어 네트워크 문제를 완화하고 전송이 성공할 수 있게 됩니다.

> git config http.postBuffer 524288000

위 명령어는 Buffer크기를 524,288,000 바이트 (500MB)로 늘리는 명령어 입니다.

해당 명령어 입력 후 git push하니 정상적으로 작동합니다!

그럼 정말 Buffer의 크기 문제였을까요? 테스트 한 번 해보죠.

## Test

> git config http.postBuffer 1000000
